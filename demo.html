🥊 moxi.js demo

Play with these in your browser. View the source (or just click each box) to see how each example is wired up — it's all inline.

1. Reactive output

<input id="name" placeholder="type your name">
<output live="this.innerText = q('#name').value ? 'hello ' + q('#name').value : ''"></output>

2. Counter with per-element state

0 
<button on-init="this.count = 0"
        on-click="this.count++; q('next output').value = this.count">click me</button>
<output>0</output>

3. Active tab with take()

selected: One

<button class="tab active" on-click="take('active', '.tab', this); ...">One</button>
<button class="tab"        on-click="take('active', '.tab', this); ...">Two</button>
<button class="tab"        on-click="take('active', '.tab', this); ...">Three</button>

4. Debounced search

<input type="search" on-input="await debounce(250); q('next output').innerText = 'searching: ' + this.value">

Only the latest keystroke after a 250ms pause produces an update.

5. View transition on toggle

The panel slides open and closed, animated by the browser's View Transition API when available. 
<button on-click="transition(() => q('#panel').classList.toggle('open'))">toggle</button>
<div id="panel">...</div>

6. Click-outside-to-dismiss

Click outside this menu to dismiss it. 
<button on-click.stop="q('#menu').hidden = false">open menu</button>
<div id="menu" hidden on-click.outside="this.hidden = true">...</div>

.stop on the opener keeps the same click from bubbling to the .outside listener, which would otherwise dismiss the menu immediately.

7. Parent listens for child emits

<div on-confirm="q('.result in this').innerText = 'got: ' + event.detail">
  <button on-click="trigger('confirm', 'yes')">yes</button>
  <button on-click="trigger('confirm', 'no')">no</button>
  <p class="result"></p>
</div>

8. Live count of a dynamic list

  • item 1
  • item 2
<button on-click="q('#items').insert('end', `<li class='item'>item ${q('.item').count + 1}</li>`)">add item</button>
<output live="this.innerText = q('.item').count + ' items'"></output>
<ul id="items">
  <li class="item">item 1</li>
  <li class="item">item 2</li>
</ul>

The count watches .item, so new items need the same class.

9. Form validity driving a button

<form id="signup">
  <input type="email" required placeholder="email">
  <input type="text" required minlength="3" placeholder="username">
  <button live="this.disabled = !q('#signup').checkValidity()">submit</button>
</form>

10. .halt on a form submit

<form on-submit.halt="q('next output').innerText = 'caught: ' + new FormData(this).get('q')">
  <input name="q" placeholder="type and press enter">
  <button type="submit">submit</button>
</form>
